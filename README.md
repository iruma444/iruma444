## Working Style

- 抽象化・構造化して、問題を扱えるサイズに分解する
- 仕様より先に「運用の現実」を置いて設計する
- "作って終わり"ではなく、改善し続けられる形にする
- AIは精度だけでなく、導入・継続利用・責任範囲まで含めて設計する
- PMとして、関係者調整・優先順位付け・進行管理まで一気通貫で担う

## Direction
